Цель проекта

  • 1

    Автоматизировать текущий процесс сбора и анализа первичной статистической
    информации с субъектов обращения фармацевтического рынка

  • 2

    Упростить и по возможности автоматизировать процесс
    подготовки отчетной документации

Решение

1

Репликация в систему Единого структурированного справочника-каталога лекарственных препаратов (ЕСКЛП) дает возможность собирать статистическую информации по стандартизированным позициям

2

Модерация форм отчетности, форматно-логический контроль и анализ полноты портфелей производителей лекарственных препаратов обеспечивают высокое качество собираемых данных

3

Визуальный конструктор форм отчетности позволяет гибко настраивать состав статистической информации, запрашиваемой у производителей лекарственных препаратов

4

Редактор сводных таблиц, графиков, дашбордов на основе Apache Superset (open source) обеспечивает построение отчетов без использования санкционных BI инструментов

Использован современный стек Веб-разработки: Django REST Framework (DRF) | React | PostgreSQL

Процесс разработки был организован в виде комбинации Rational Unified Process (RUP) и гибких методологий разработки ПО

ЭФФЕКТ от использования

Система внедрена в промышленную эксплуатацию на федеральном уровне

Полностью автоматизирован процесс сбора информации от респондентов

Полностью автоматизировано построение стандартных отчетов

Реализован BI-инструмент для построения нестандартных отчетов без написания кода
